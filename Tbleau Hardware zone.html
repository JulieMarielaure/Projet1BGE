Hex to binary is very easy already, so much easier than hex to decimal or binary to decimal. This is because each HEX character is exactly 4 binary numbers so converting is just a matter of grouping binary numbers into 4s of translating each HEX character into 4 binary numbers.

in case you didn't know la k,

3Fh => [3] [F] => [0011] [1111]

0101 1101 => [0101] [1101] => [5] [D] => 5Dh

don't tell me you can't easily get used to the first sixteen numbers, 0H - FH

To convert hex to decimal you just learn counting...
0 >> 0, 1 >> 1 .... 9 >> 9, 10 >> A, 11 >> B... 15 >> F.

To convert binary to decimal you just do addition of (8,4,2,1) for the bits that you have.
0111 >> (0*8+1*4+1*2+1*1=7), 1010 >> (1*8+0*4+1*2+0*1=10)

With the intermediate decimal step you should be able to remember these quite easily.

decimal value of Nth bit in base B = B^(n-1)
for binary (base 2) = 2^(n-1)
for hexadecimal (base 16) = 16^(n-1)

Code: binary 0101 = 0*2^3 + 1*2^2 + 0*2^1 + 1*2^0 = 0 + 4 + 0 + 1 = decimal 5 1010 = 1*2^3 + 0*2^2 + 1*2^1 + 0*2^0 = 8 + 0 + 2 + 0 = decimal 10 hexadecimal AB = 10*16^1 + 11*16^0 = 160 + 11 = 171 hexadecimal FE = 15*16^1 + 14*16^0 = 240 + 14 = 254
simple way to convert hex to binary is to convert each digit into the binary equivalent (each hex digit can be expressed as 4 binary digits):

e.g:
Code: AB = 1010 1011 = 2^7 + 2^5 + 2^3 + 2^1 + 2^0 = 128 + 32 + 8 + 2 + 1 = 171
Most engineers and developers can remember the binary equivalent of hexadecimals by heart.
